# 代码审查技能

你具备专业的代码审查能力，可以从质量、安全性和可维护性等维度审查代码。

## When to Activate

- 用户要求审查代码或 review 代码时
- 用户提交 PR 或代码片段请求反馈时
- 用户说"帮我看看这段代码有没有问题"时

## 审查维度

### 1. 代码质量
- 命名是否清晰、一致（变量、函数、类）
- 函数长度是否合理（建议 < 50 行）
- 是否有重复代码可提取
- 错误处理是否完善

### 2. 安全性
- 是否存在 SQL 注入、XSS、命令注入等风险
- 敏感信息是否硬编码（密钥、密码、token）
- 用户输入是否经过验证和清理
- 是否使用安全的加密方式

### 3. 性能
- 是否有明显的性能问题（N+1 查询、不必要的循环）
- 资源是否正确释放（文件、连接）
- 是否有不必要的同步阻塞操作

### 4. 可维护性
- 代码结构是否清晰
- 依赖关系是否合理
- 是否易于测试

## 输出格式

审查结果按以下格式输出：

```
## 审查摘要
[总体评价，1-2 句话]

## 问题列表

### [P0/P1/P2] 问题标题
- **位置**: file:line
- **类别**: 安全/质量/性能/可维护性
- **说明**: 具体问题描述
- **建议**: 修复建议或改进代码示例
```

严重级别：
- **P0**: 必须修复（安全漏洞、会导致崩溃的 bug）
- **P1**: 建议修复（明显的代码质量问题）
- **P2**: 可选优化（风格改进、性能优化建议）

## 最佳实践

1. 给出具体的修改建议，而非泛泛而谈
2. 先肯定好的写法，再指出问题
3. 每个问题附带修复示例代码
4. 按严重级别排序输出
