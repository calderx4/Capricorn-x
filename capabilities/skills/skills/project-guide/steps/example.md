# 示例：完整协作流程参考

> 一个通用项目的对话范例。展示每个步骤中用户和 Capricorn 的典型交互模式。
> 垂类可基于此范例替换为自己的领域场景。

---

## 1.1 理解需求

> **用户**：我这边有一批数据，想让你帮忙处理一下，主要是分类整理、找出异常、生成报告。

> **Capricorn**：好的，我先确认几个问题：
> 1. 数据在哪？什么格式？
> 2. "异常"具体指什么？缺失？重复？偏离？
> 3. 结果给谁看？需要什么格式？

> **用户**：文件在 workspace/main/ 下面，CSV 格式。异常就是偏离正常范围的。结果给团队看，要有对比和重点。

> **Capricorn**：明白了，我先看一下数据结构。

*(查看后写 requirements.md，用户确认方向)*

---

## 1.2 试做样本

> **Capricorn**：我先拿第一批数据做一份样本，你看看效果。

*(执行后展示)*

> **用户**：分类光列了个表，没有说明。也没有和上一期对比，看不出变化。偏离的也没标出来。

> **Capricorn**：收到。记下来：
> - 分类要有简要说明
> - 时间维度必须有前后对比
> - 偏离的值要标记
>
> 写到 alignment.md，再做一份改进版。

*(改进版用户认可)*

---

## 1.3 建能力

*(检查现有工具是否够用，缺什么补什么)*

---

## 1.4 定质量标准

> **Capricorn**：根据你的反馈，我整理了质量标准：
> - 有标题结构：至少 1 个 Markdown 标题
> - 有具体数字：至少 2 个
> - 内容充分：至少 100 字符
>
> 写到 alignment.md 质量底线了。垂类可以在此基础上增加领域特有维度。

> **用户**：行。

---

## 1.5 协商自进化策略

> **Capricorn**：正式跑之前，确认一下自进化的边界：
> 1. 连续失败几次开始纠偏？（建议 3 次）
> 2. 能改什么？只改行为规则？还是也能改执行步骤？
> 3. 自动修改直接生效？还是等你回来确认？

> **用户**：阈值 3 次，行为规则和执行步骤都可以改，新建工具要等我确认，自动修改直接生效。

> **Capricorn**：好，写到 evolution_policy.md。

---

## 1.6 拆任务

> **Capricorn**：我拆成这些任务：
> 1. {核心任务1} — 高优先级
> 2. {核心任务2} — 高优先级
> 3. {关联任务} — 高优先级
> 4. {异常检测} — 中优先级
> 5. {汇总产出} — 中优先级
>
> 写到 tasks.md 了。

> **用户**：没问题，开始吧。

---

## 创建 Cron → Phase 2

> **Capricorn**：创建两个 Cron：
> - 执行 Cron（每小时）：从 tasks.md 领任务执行
> - 验证 Cron（每4小时）：quality_check + 纠偏
>
> 设置好了，我会自主执行，有问题随时找我。

---

## Phase 2 自动运行（几天后）

```
Day1 09:00  executor → 任务1 → 产出 → 自检通过
Day1 13:00  executor → 任务2 → 产出 → 自检通过
Day1 16:00  verifier → 检查两份 → 都通过 → quality_signal record
Day2 09:00  executor → 任务3 → 产出
Day2 13:00  verifier → 任务3缺对比 → 不通过 → 写修改意见
Day2 14:00  executor → 读修改意见 → 补充 → 重新提交
Day2 16:00  verifier → 通过 → quality_signal record
Day3 09:00  executor → 任务4 → 产出
...
```

纠偏示例（连续 3 次缺标题）：
```
verifier 发现 has_headings 连续 3 次不通过
  → bia_update: "产出必须包含至少一个 Markdown 标题"
  → changelog 记录
  → 下次 executor 自动加标题
```

---

## Phase 3 人工介入

> **用户**：最近进展怎么样？

> **Capricorn**：汇报：
> - 任务进度：5/5 完成
> - 质量通过率：82%
> - 自动修改 1 条：bia.md 追加对比规则
>
> 这个自动修改你可以吗？

> **用户**：可以。帮我加个新任务。

> **Capricorn**：加到 tasks.md 了，executor 下次会自动领取。

---

## 用户全程只做了 4 件事

1. 描述需求
2. 看样本给反馈
3. 确认标准和策略
4. 随时回来查看进展

其余全是 Capricorn 自主完成。
