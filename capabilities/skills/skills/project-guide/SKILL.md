# Project Guide

## 你应该怎么做

### 新对话开始时
1. 检查工作区是否有 `.phase` 文件
2. **无文件** → 用户是新来的，主动问："你好，你想做什么？我来帮你规划和执行。"
3. **有文件** → 读取当前阶段，按下面对应阶段的行为引导

### 阶段1：项目启动（人机协作）
你的任务是把项目从零搭建到可以自主运行。按顺序做：

1. **理解需求** → 问清楚：信息源在哪？产出给谁看？重点关注什么？→ 写 `requirements.md`
2. **试做样本** → 挑一个最典型任务跑一次 → 给用户看结果 → 收集反馈 → 写 `alignment.md`
3. **建能力** → 检查现有工具够不够，缺什么补什么（tool/skill/workflow）
4. **定质量标准** → 把用户反馈提炼为可检查的规则 → 更新 `alignment.md` 质量底线
5. **协商自进化策略** → 问用户：连续失败几次纠偏？能改什么？需不需要确认？→ 写 `evolution_policy.md`
6. **拆任务** → 把需求拆成可执行的子任务 → 写 `tasks.md`

每一步做完都要展示给用户确认，再进入下一步。
详细步骤参考：`steps/01-bootstrap.md`

### 阶段2：自主执行（Capricorn 全权）
当阶段1 退出条件全部满足后：
1. 创建 executor cron（每小时领取任务执行）和 verifier cron（定期验证质量）
2. 告诉用户："设置好了，我会自主执行，有问题随时找我。"
3. 进入静默运行，不打扰用户

详细步骤参考：`steps/02-autonomous.md`

### 阶段3：人工介入（用户回来了）
用户主动发起对话时：
1. **先汇报**：任务进度 + 质量汇总 + 自动改动列表
2. **逐个确认**：自动修改的规则要不要保留？
3. **问新需求**：有没有新的方向或调整？
4. 对齐完成后回到阶段2 继续

详细步骤参考：`steps/03-intervention.md`

## 阶段1 退出条件

- [ ] `requirements.md` — 项目需求
- [ ] `alignment.md` — 对齐规则
- [ ] `evolution_policy.md` — 自进化策略
- [ ] `tasks.md` — 任务列表
- [ ] 至少 1 个 tool/skill/workflow 已创建

## 阶段转换

```
1 → 2：退出条件全部满足
2 → 3：用户主动介入
3 → 2：对齐完成，回到执行
```

每次转换时更新 `.phase` 文件。详细交互范例见 `steps/example.md`（按需加载）。

## 用户犹豫"能不能放手"

当用户问"什么时候可以自动跑"、"做到什么程度够了"、"要不要再完善一下"时，加载 `steps/automation-readiness.md`。核心原则：**先跑起来，边跑边改。**
